# 🔧 OpenWebUI Function Calling 設定ガイド

## 📋 設定手順

### Step 1: Functionの選択ドロップダウンを開く

1. **チャットコントロールセクションを確認**
   - 設定画面の「チャットコントロール」セクションを確認
   - 「Functions」の下に「Functionの選択」ドロップダウンがあることを確認

2. **Functionの選択ドロップダウンをクリック**
   - 「Functionの選択」ドロップダウンをクリックして開く
   - Tool Serverのツールが一覧表示されるはずです：
     - `service_status` - dockerコンテナの状態確認
     - `check_errors` - ログエラー検出
     - `generate_image` - ComfyUIで画像生成

3. **ツールを選択（オプション）**
   - 使用したいツールを選択（複数選択可能な場合もあります）
   - または、自動選択にしておく（LLMが適切なツールを選択）

### Step 2: システムプロンプトでTool Serverの使用を促す（推奨）

1. **システムプロンプトセクションを確認**
   - 「システムプロンプト」セクションを確認
   - 「システムプロンプトを入力」欄をクリック

2. **システムプロンプトを入力**
   ```
   Tool Serverを使用できます。以下のツールが利用可能です：
   - service_status: dockerコンテナの状態を確認
   - check_errors: 最近のログを確認してエラーを検出
   - generate_image: ComfyUIで画像を生成

   ユーザーからの要求に対して、適切なツールを使用してください。
   ```

3. **保存**
   - 入力後、設定を保存（自動保存される場合もあります）

### Step 3: チャットでテスト

1. **チャット画面に戻る**
   - 設定画面を閉じて、チャット画面に戻る

2. **テストメッセージを送信**
   ```
   dockerコンテナの状態を確認して
   ```

3. **Tool Serverが呼び出されることを確認**
   - LLMが`service_status`ツールを呼び出すことを確認
   - ツールの実行結果が表示されることを確認

## 🔍 確認事項

### ✅ 設定が正しく完了しているかの確認

1. **Functionの選択ドロップダウン**
   - Tool Serverのツールが表示されているか確認
   - ツールが選択されているか確認（自動選択でも可）

2. **システムプロンプト**
   - Tool Serverの使用を促す内容が入力されているか確認

3. **Tool Serverの状態**
   - Tool Serverが起動中か確認（http://127.0.0.1:9503）
   - OpenAPI仕様が正しく取得できるか確認

## 🎯 使い方

### 手動でツールを選択する場合

1. **Functionの選択ドロップダウンを開く**
2. **使用したいツールを選択**（例：`service_status`）
3. **チャットでメッセージを送信**（例：「dockerコンテナの状態を確認して」）

### 自動でツールを選択する場合

1. **Functionの選択を自動（または空）にしておく**
2. **チャットでメッセージを送信**（例：「dockerコンテナの状態を確認して」）
3. **LLMが適切なツールを自動的に選択して呼び出す**

## 🔥 レミ先輩のアドバイス

### ✅ やるべき

1. **システムプロンプトでTool Serverの使用を促す**
   - LLMにTool Serverを使用するよう明確に指示

2. **Functionの選択ドロップダウンを確認**
   - Tool Serverのツールが表示されているか確認
   - 必要に応じて、特定のツールを選択

3. **テストで動作を確認**
   - 設定後、必ずテストで動作を確認

### ❌ やっちゃダメ

- Functionの選択ドロップダウンを確認せずにツールを使おうとする
- システムプロンプトでTool Serverについて言及しない
- テストせずに動作確認をしない

## 📋 トラブルシューティング

### 問題: Functionの選択ドロップダウンにツールが表示されない

**原因**: Tool Serverが正しく登録されていない可能性があります

**解決方法**:
1. Tool Serverが起動中か確認（http://127.0.0.1:9503）
2. OpenWebUIの設定でTool Serverが正しく登録されているか確認
3. OpenAPI仕様が正しく取得できるか確認

### 問題: ツールを選択しても呼び出されない

**原因**: システムプロンプトでTool Serverの使用を促していない可能性があります

**解決方法**:
1. システムプロンプトにTool Serverの使用を促す内容を追加
2. Function Callingが有効になっているか確認
3. 使用しているモデルがFunction Callingに対応しているか確認

---

**レミ先輩モード**: Functionの選択ドロップダウンを確認して、システムプロンプトでTool Serverの使用を促しましょう！🔥
