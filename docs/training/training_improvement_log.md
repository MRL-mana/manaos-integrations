## 実施した改善

### 1. Gradient Checkpointingの有効化
- **目的**: VRAM節約
- **効果**: アクティベーションのメモリ使用量を削減
- **実装**: `model.gradient_checkpointing_enable()` を追加

### 2. 低メモリ設定の実装
- **バッチサイズ**: 2 → 1
- **最大シーケンス長**: 2048 → 1024
- **Gradient Accumulation**: 4（実効バッチサイズ: 4）
- **効果**: VRAM要件を約50%削減

### 3. メモリ効率の向上
- **`low_cpu_mem_usage=True`**: RAM節約
- **`optim="adamw_torch"`**: メモリ効率の良いオプティマイザ

### 4. 監視スクリプトの作成
- **`check_memory_status.py`**: VRAM/RAM使用状況の詳細確認
- **`check_training_progress.py`**: 学習進行状況の詳細確認
- **`monitor_training.py`**: 学習監視

python train_castle_ex_full.py \

  --epochs 25 \

## トラブルシューティング

### エラーが発生した場合

1. **Out of Memory (OOM)**
   - バッチサイズを1に減らす（既に適用済み）
   - 最大シーケンス長を512に減らす
   - Gradient Accumulationを8に増やす

2. **モデル読み込みエラー**
   - キャッシュをクリア
   - モデルを再ダウンロード
   - CPUオフロードを検討

3. **データ前処理エラー**
   - データファイルの整合性を確認
   - メモリ不足の場合はバッチ処理を検討
