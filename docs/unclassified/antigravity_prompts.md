## 🔥 MOC再構築プロンプト

### 基本MOC作成

```
以下のノートを1本のMOC（Map of Content）に再構築して。

【要件】
- 関連するノートをグループ化
- 各グループに簡潔な説明を追加
- ノート間の関係性を明確に
- Obsidianのリンク形式（[[ノート名]]）を使用

【出力形式】
- Markdown形式
- YAMLフロントマター付き
- 見出し構造: # MOC名 → ## グループ名 → ### ノート名

【ノート】
（ここにノートを貼り付け）
```

---

### テーマ別MOC作成

```
以下のノートを「[テーマ名]」というテーマでMOCに再構築して。

【要件】
- テーマに関連するノートだけを選別
- 重要度順に並べる
- 各ノートの要点を1行で要約
- ノート間の依存関係を明確に

【出力形式】
- Markdown形式
- YAMLフロントマター付き（tags: [moc, [テーマ名]]）
- 見出し構造: # MOC: [テーマ名] → ## 基本 → ## 発展 → ## 参考

【ノート】
（ここにノートを貼り付け）
```

## 🎨 YAML一括編集プロンプト

### タグの追加

```
以下のノートのYAMLフロントマターに、適切なタグを追加して。

【要件】
- 既存のタグは保持
- 内容に基づいてタグを追加
- タグは小文字、ハイフン区切り
- 最大5個まで

【タグ候補】
- daily, health, idea, learning, review, moc
- manaos, notebooklm, antigravity
- [テーマ名]

【ノート】
（ここにノートを貼り付け）
```

## 🎯 カスタムプロンプトの作り方

### 良いプロンプトの特徴

1. **要件を明確に**: 「【要件】」セクションを使う
2. **出力形式を指定**: 「【出力形式】」セクションを使う
3. **具体例を入れる**: 見出し構造を指定
4. **制約を明確に**: 「最大5個まで」など

### プロンプトテンプレート

```
以下のノートを[目的]に再構築して。

【要件】
- [要件1]
- [要件2]
- [要件3]

【出力形式】
- Markdown形式
- YAMLフロントマター付き（[メタデータ]）
- 見出し構造: [構造]

【ノート】
（ここにノートを貼り付け）
```

## 📋 プロンプトの使い方

### STEP 1: ノートを選択

1. Obsidianから関連するノートを複数選択
2. 内容をコピー

### STEP 2: Antigravityで実行

1. Antigravityを開く
2. プロンプトを選択（またはカスタム）
3. ノートを貼り付け
4. 実行

### STEP 3: 結果を保存

1. 生成されたMarkdownをコピー
2. Obsidianに保存
3. ManaOSに連携（必要に応じて）

---

## 🔧 ManaOS連携設定

### 自動化ワークフロー（n8n）

1. **週次MOC作成**
   - 毎週日曜日の夜、関連ノートを自動選択
   - Antigravity APIに送信（または手動でコピペ）
   - 結果を `MOCs/` に保存
   - ManaOSの記憶システムに登録

2. **記事自動生成**
   - 学習ログが一定量たまったら自動実行
   - Antigravityで記事化
   - note/Zennに自動投稿（オプション）
