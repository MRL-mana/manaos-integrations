# 🚀 サーバー起動の簡単な方法

## ⚠️ ヘッドレスモードがうまく動作しない場合

ヘッドレスモード（「ローカル LLM サービスを有効にする」チェックボックス）は、設定によっては自動的にサーバーが起動しない場合があります。

---

## ✅ 最も確実な方法：GUIから直接起動

### ステップ1: 「Server」タブを見つける

**方法A: 左サイドバーから**
1. 画面の**左側**にある縦に並んだアイコンを確認
2. 「Server」アイコン（🖧）を探す
3. クリック

**方法B: 上部のタブメニューから**
1. 上部のタブメニューを確認
2. 「Server」タブを探す
3. クリック

**方法C: 検索機能を使う**
1. LM Studioの検索機能（Ctrl+F または Cmd+F）を使う
2. 「Server」と検索

### ステップ2: モデルを選択

1. 「**Select a model to load**」をクリック
2. ダウンロード済みのモデルを選択
   - 推奨: Qwen2.5-Coder-7B-Instruct
   - または: DeepSeek-Coder-6.7B-Instruct

### ステップ3: 「Start Server」をクリック

1. 「**Start Server**」ボタンを探す
2. **クリック**
3. サーバーが起動すると、ボタンが「Stop Server」に変わります

---

## 🔍 サーバーが起動したか確認

### 方法1: ブラウザで確認

ブラウザで以下にアクセス：
- `http://localhost:1234/v1/models`

**期待される結果：**
- JSON形式でモデル一覧が表示される
- エラーページが表示される場合は、サーバーが起動していません

### 方法2: PowerShellで確認

```powershell
.\check_running_status.ps1
```

または：

```powershell
Invoke-WebRequest -Uri "http://localhost:1234/v1/models" -Method GET
```

## 💡 トラブルシューティング

### 「Server」タブが見つからない

1. **LM Studioのバージョンを確認**
   - バージョンが古い可能性があります
   - 最新版に更新: https://lmstudio.ai/download

2. **LM Studioを再起動**
   - アプリを完全に閉じて再起動

3. **左サイドバーが折りたたまれている**
   - 画面の左端にマウスを合わせる
   - または、ウィンドウの左端を確認

### 「Start Server」ボタンが押せない

- **モデルが選択されていない**
  - 先に「Select a model to load」でモデルを選択

- **モデルがダウンロードされていない**
  - 「Search」タブでモデルを検索してダウンロード

### サーバーが起動しない

- **ポート1234が使用されている**
  - 他のアプリケーションがポート1234を使用している可能性
  - LM Studioを再起動

- **ファイアウォールの設定**
  - ローカル接続がブロックされていないか確認

---

## 🎯 推奨手順

1. **「Server」タブを開く**（左サイドバーまたは上部タブメニューから）
2. **モデルを選択**（「Select a model to load」から）
3. **「Start Server」をクリック**
4. **起動確認**（ブラウザまたはPowerShellで）

---

**GUIから直接「Start Server」をクリックする方法が最も確実です！**
