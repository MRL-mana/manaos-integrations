# GPU/CPU自動フォールバック機能

**実装日**: 2025-12-28

---

## 🎯 機能概要

GPUが他のプロセス（Stable Diffusion、他のLLM推論など）で使用されている場合、自動的にCPUモードに切り替えてLLM推論を実行します。

---

## 🔧 実装内容

### 1. GPU使用状況の検出

- **Windows環境**: 複数のollamaプロセスを検出
- **Linux環境**: nvidia-smiでGPU使用率をチェック（50%以上でCPUモード）

### 2. CPUモードへの自動切り替え

- Ollama APIの`num_gpu: 0`パラメータでCPUモードを指定
- GPU使用中を検出した場合、自動的にCPUモードで実行

### 3. ログ記録

- CPUモードで実行した場合、ログに記録
- レスポンスに`cpu_mode: true`フラグを追加

---

## 📊 動作フロー

```
1. LLMルーティング要求
   ↓
2. GPU使用状況をチェック
   ↓
3. GPU使用中？
   ├─ Yes → CPUモード（num_gpu: 0）で実行
   └─ No  → GPUモード（デフォルト）で実行
   ↓
4. 結果を返す（cpu_modeフラグ付き）
```

---

## 💡 メリット

1. **リソース競合の回避**: GPUが使用中でもLLM推論が実行可能
2. **自動切り替え**: 手動設定不要
3. **透明性**: ログでCPU/GPUモードを確認可能

---

## ⚠️ 注意事項

### CPUモードの制限

- **処理速度**: GPUモードより大幅に遅い（数倍〜数十倍）
- **推奨用途**: 
  - 軽量な会話タスク（`conversation`）
  - 緊急時のフォールバック
- **非推奨**: 
  - 重い推論タスク（`reasoning`、特に72Bモデル）
  - リアルタイム応答が必要な場合

### 推奨設定

夜のルーチンなど重いタスクは、以下のいずれかを推奨：

1. **モデルの軽量化**: `qwen2.5:32b`や`qwen2.5:14b`を使用
2. **タイムアウトの延長**: 90秒以上に設定
3. **非同期実行**: バックグラウンドで実行

---

## 🔍 ログ例

```
[LLM Routing] reasoning -> qwen2.5:72b (primary, CPU) | latency: 45230ms | fallback: False
```

- `(primary, CPU)`: primaryモデル、CPUモードで実行
- `latency: 45230ms`: 処理時間（CPUモードは遅い）
