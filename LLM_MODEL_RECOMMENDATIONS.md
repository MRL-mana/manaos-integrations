## 💡 ベストプラクティス

### 現在の設定（動的ロード）がベストな理由

1. **メモリ効率**
   - 使用していないモデルは自動アンロード
   - メモリ使用量を最小化

2. **柔軟性**
   - 用途に応じて最適なモデルを選択
   - 複数のモデルを効率的に使い分け

3. **パフォーマンス**
   - 軽量タスクには軽量モデル
   - 重いタスクには大型モデル
   - レスポンス時間を最適化

### 常時ロード推奨ケース

以下の場合は常時ロードを検討：

1. **高頻度で使用するモデル**
   - 例: `llama3.2:3b`（会話用）
   - ロード時間を削減

2. **低レイテンシが必要**
   - 例: リアルタイムチャット
   - 即座に応答が必要

3. **メモリに余裕がある**
   - 例: 32GB以上のRAM
   - 複数モデルを常時ロード可能

## 🔧 設定変更方法

### 常時ロードに変更する場合

```yaml
# docker-compose.always-ready-llm.yml
environment:
  - OLLAMA_KEEP_ALIVE=24h  # 24時間保持
  - OLLAMA_MAX_LOADED_MODELS=3  # 最大3モデル同時ロード
```

### 推奨設定（現在の設定）

```yaml
environment:
  - OLLAMA_KEEP_ALIVE=5m  # 5分間保持
  - OLLAMA_MAX_LOADED_MODELS=2  # 最大2モデル同時ロード
```
