# 🔧 ツールが表示されない場合の高度なトラブルシューティング

## ❌ 問題

「ツールの選択」ドロップダウンにTool Serverのツールが表示されない。

## 🔍 Web検索で見つかった追加の解決策

### 1. ブラウザのキャッシュクリアと再起動

**原因**: ブラウザのキャッシュが原因で表示されない場合があります。

**解決方法**:
1. **ブラウザのキャッシュをクリア**
   - Chrome: `Ctrl + Shift + Delete` → キャッシュされた画像とファイルを選択 → クリア
   - Edge: `Ctrl + Shift + Delete` → キャッシュされた画像とファイルを選択 → クリア
   - Firefox: `Ctrl + Shift + Delete` → キャッシュを選択 → クリア

2. **ブラウザを完全に再起動**
   - すべてのタブを閉じる
   - ブラウザを完全に終了
   - ブラウザを再起動

3. **OpenWebUIにアクセス**
   - `http://localhost:3001` にアクセス
   - 「ツールの選択」ドロップダウンを確認

### 2. OpenWebUIの再起動

**原因**: 設定変更後、OpenWebUIが変更を認識していない可能性があります。

**解決方法**:
```powershell
# OpenWebUIコンテナを再起動
docker-compose -f docker-compose.always-ready-llm.yml restart openwebui

# 再起動後、少し待ってから確認
Start-Sleep -Seconds 5

# アクセス確認
Invoke-WebRequest -Uri "http://localhost:3001" -UseBasicParsing -TimeoutSec 5
```

### 3. 拡張機能の影響確認

**原因**: 他の拡張機能との競合が原因でツールが表示されないことがあります。

**解決方法**:
1. **ブラウザの拡張機能を一時的に無効化**
   - Chrome: `chrome://extensions/`
   - Edge: `edge://extensions/`
   - Firefox: `about:addons`

2. **問題が解消するか確認**
   - 拡張機能を無効化した状態でOpenWebUIにアクセス
   - 「ツールの選択」ドロップダウンを確認

3. **問題が解消した場合**
   - 拡張機能を1つずつ有効化して、原因を特定

### 4. OpenAPI仕様の構造確認

**確認済み**: OpenAPI仕様は正常に取得できています。

**確認内容**:
- ✅ タイトル: manaOS Tool Server
- ✅ バージョン: 1.0.0
- ✅ エンドポイント数: 5個（3つのツールエンドポイントを含む）
- ✅ operationId: 正常に設定されている

### 5. Function Callingの設定確認

**確認済み**: Function Callingは「ネイティブ」モードに設定されています。

**追加確認**:
- 「高度なパラメータ」セクションで「Function呼び出し」が「ネイティブ」になっているか確認
- 「デフォルト」になっている場合は「有効」または「自動」に変更

### 6. Tool Serverの接続状態確認

**確認方法**:
1. OpenWebUIの設定画面を開く（⚙️ → External Tools）
2. Tool Server（`http://localhost:9503`）の接続状態を確認
3. 「Connected」になっているか確認
4. 接続されていない場合、再度登録

### 7. ブラウザの開発者ツールで確認

**確認方法**:
1. **ブラウザの開発者ツールを開く**（F12）
2. **Networkタブを開く**
3. **「ツールの選択」ドロップダウンを開く**
4. **以下のリクエストを確認**:
   - `/api/v1/tools/` へのリクエスト
   - `http://localhost:9503/openapi.json` へのリクエスト
5. **エラーメッセージがないか確認**:
   - Consoleタブでエラーメッセージを確認
   - Networkタブで失敗したリクエストを確認

## 🔥 レミ先輩の推奨順序

### 優先度1: ブラウザのキャッシュクリアと再起動
- **理由**: 最も簡単で効果的な解決策
- **実行時間**: 2分

### 優先度2: OpenWebUIの再起動
- **理由**: 設定変更を確実に反映
- **実行時間**: 1分

### 優先度3: ブラウザの開発者ツールで確認
- **理由**: エラーの原因を特定できる
- **実行時間**: 3分

### 優先度4: 拡張機能の影響確認
- **理由**: 他の原因が特定できない場合
- **実行時間**: 5分
