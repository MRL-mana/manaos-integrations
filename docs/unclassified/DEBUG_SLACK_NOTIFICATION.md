# 🔍 Slack通知が届かない問題の診断

## 🎯 確認手順

### Step 1: n8nのワークフロー実行履歴を確認

1. **n8nを開く**: http://localhost:5678
2. **「Executions」タブを開く**（左メニュー）
3. **最新の実行を確認**:
   - Browse AI Webhookノードでデータを受信しているか
   - エラーが発生していないか

---

### Step 2: ワークフローが有効化されているか確認

1. **ワークフローを開く**: Browse AI統合ワークフロー
2. **右上のトグルスイッチがONになっているか確認**
3. **OFFの場合はONにする**

---

### Step 3: 重要度スコアを確認

ワークフローのコードでは、**重要度スコアが5以上**の場合のみSlackに通知します。

**現在のロボット**: 「トレンドリポジトリリスト」

**重要度スコア計算ロジック**:
- Trendingロボットの場合:
  - `stars`または`starCount`があれば: +5点
  - `trending`があれば: +5点
  - **合計10点**（通知されるはず）

**問題の可能性**:
- Browse AIから送られてくるデータの形式が想定と異なる
- `extractedData.stars`ではなく、別のフィールド名になっている可能性

---

### Step 4: データ形式を確認

Browse AIから送られてくるデータの実際の形式を確認する必要があります。

**確認方法**:
1. n8nの「データ整形・重要度判定」ノードをクリック
2. **「Execute step」をクリック**
3. **出力データを確認**:
   - `importance`の値が5以上か
   - `shouldNotify`が`true`か

---

## 🔧 即座に試せる修正

### 修正1: 重要度スコアの閾値を下げる

現在の閾値（5点）が高すぎる可能性があります。

**修正方法**:
1. n8nの「通知判定」ノードを開く
2. 条件を変更:
   ```
   {{ $json.shouldNotify }} == true
   ```
   または、重要度スコアを直接確認:
   ```
   {{ $json.importance }} >= 0
   ```
   （すべての通知を送る）

---

### 修正2: データ形式を確認して修正

Browse AIから送られてくるデータの実際の形式を確認して、コードを修正します。

**確認手順**:
1. n8nの「Browse AI Webhook」ノードをクリック
2. **「Execute step」をクリック**
3. **出力データを確認**:
   - `body`の中身を確認
   - `extractedData`の構造を確認

---

### 修正3: テスト用に強制通知

一時的に、すべての通知を送るように設定します。

**修正方法**:
1. n8nの「通知判定」ノードを開く
2. 条件を変更:
   ```
   {{ true }}
   ```
   （常にtrueにして、すべての通知を送る）

---

   - 通知判定ノード: 条件を満たしているか

---

2. **n8nの実行履歴を確認**

## 💡 よくある問題と解決策

### 問題1: ワークフローが有効化されていない

**解決策**: ワークフローの右上のトグルスイッチをONにする

---

### 問題2: 重要度スコアが5未満

**解決策**: 
- 閾値を下げる（例: `>= 0`）
- または、データ形式を確認してスコア計算ロジックを修正

---

### 問題3: Browse AIからWebhookが呼ばれていない

**確認方法**:
- n8nの実行履歴に新しい実行が記録されているか
- ngrokが起動しているか（`http://localhost:4040`で確認）

**解決策**:
- ngrokを再起動
- Browse AIのWebhook URLを再確認

---

### 問題4: Slack Webhook URLが無効

**確認方法**:
- PowerShellで直接テスト:
```powershell
$webhookUrl = "https://hooks.slack.com/services/<YOUR>/<WEBHOOK>/<URL>"
$body = @{ text = "テスト" } | ConvertTo-Json
Invoke-RestMethod -Uri $webhookUrl -Method Post -Body $body -ContentType "application/json"
```
