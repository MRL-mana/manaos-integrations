# 外から 200 / 401 / 429 を取る手順（30秒）

**どこでやる**: **母艦のターミナルでやる**。このはサーバーは使わない。
- 母艦だけで構成する場合: Gateway を母艦で起動し、`http://127.0.0.1:8088` 向けに 200 だけ確認すればOK（401/429 は Nginx を使うとき用）。
- このはで Gateway を公開している場合: 母艦からそのドメインに curl して 200/401/429 を取る。

**事前に用意する**:
- ドメイン（例: `mrl-mana.com`）… このはのIPに向いていること
- Basic認証のユーザー名とパスワード（htpasswd で作ったやつ）

**一発で取る**: 母艦のターミナルで同じフォルダにいる状態で:

- **PowerShell**:
  `.\check_external_200_401_429.ps1 -Domain "mrl-mana.com" -User "mana" -Pass "あなたのパスワード"`
- **bash**:
  `DOMAIN=mrl-mana.com USER=mana PASS=xxx ./check_external_200_401_429.sh`

出た結果をそのまま貼ればOK。

---

## ① 200 を取る（health・認証なし）

ターミナルで（`<domain>` を自分のドメインに置き換え）:

```bash
curl -I https://<domain>/moltbot/health
```

**期待**: 1行目が `HTTP/2 200` または `HTTP/1.1 200`

**貼るもの**: 1行目だけでOK。例: `HTTP/2 200`

---

## ② 401 を取る（/moltbot/ は認証必須）

認証**なし**で `/moltbot/` を叩く:

```bash
curl -I https://<domain>/moltbot/ -s -o /dev/null -w "%{http_code}\n"
```

**期待**: 出てくる数字が `401`

**貼るもの**: 出た数字だけ。例: `401`

---

## ③ 429 を取る（レート制限）

認証**あり**で短時間に連打する（`<domain>` / `<user>` / `<pass>` を置き換え）:

```bash
for i in {1..30}; do
  curl -s -o /dev/null -u <user>:<pass> -w "%{http_code}\n" https://<domain>/moltbot/
done | sort | uniq -c
```

**期待**: `429` が混ざる（全部 200 でも 429 が1回でも出ればOK）

**貼るもの**: コマンドの出力そのまま。例:
```
     25 200
      5 429
```

---

## 返すとき

この3つをそのまま貼ればOK:

1. ①の1行目（`HTTP/2 200` など）
2. ②の数字（`401`）
3. ③の `sort | uniq -c` の結果（`25 200` と `5 429` のような行）

**Windows（PowerShell）で ③ やる場合**:

```powershell
1..30 | ForEach-Object {
  (Invoke-WebRequest -Uri "https://<domain>/moltbot/" -Credential (Get-Credential) -UseBasicParsing).StatusCode
} | Group-Object | Select-Object Count, Name
```

または WSL で上の `for i in {1..30}` のコマンドをそのまま実行してもよい。

---

## 詰まったとき

- **200 が出ない** → DNS・ファイアウォール・Certbot・`server_name` を確認（`CHECKLIST_A_EXTERNAL.md` の「外部疎通でよくある詰まり」参照）
- **401 にならない** → Nginx の `location = /moltbot/health`（認証なし）と `location /moltbot/`（認証あり）の優先順位を確認
- **429 が出ない** → `limit_req_zone` を http ブロックに書いたか、rate/burst を確認
