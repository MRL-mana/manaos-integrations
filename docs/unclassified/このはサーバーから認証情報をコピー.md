# このはサーバーからGoogle Drive認証情報をコピー

## 🎯 概要

このはサーバー側で既に使用しているGoogle Drive認証情報（credentials.json、token.json）を母艦（新PC）にコピーして使用する方法。

---

## ✅ メリット

- 新規に認証情報を作成する必要がない
- 既存の認証情報をそのまま使用可能
- 時間短縮（約5分→約1分）

---

## 🚀 方法1: 自動コピースクリプトを使用（推奨）

```powershell
cd C:\Users\mana4\OneDrive\Desktop\manaos_integrations
.\copy_google_drive_from_konoha.ps1
```

このスクリプトが以下を自動実行:
- このはサーバーへの接続確認
- 認証情報ファイルの検索
- ファイルのコピー

---

## 🔧 方法2: 手動でコピー

### ステップ1: このはサーバーに接続

```powershell
# 直接SSH（SSHサービスが復旧している場合）
ssh konoha

# またはX280経由
ssh konoha-via-x280

# またはTailscale IP直接
ssh root@100.93.120.33
```

### ステップ2: 認証情報ファイルの場所を確認

```bash
# このはサーバー側で実行
find /root -name "credentials.json" -o -name "token.json" 2>/dev/null

# または一般的な場所を確認
ls -la /root/credentials.json
ls -la /root/token.json
ls -la /root/.mana_vault/credentials.json
ls -la /root/.mana_vault/token.json
```

### ステップ3: ファイルをコピー

#### 方法A: scpコマンドを使用

```powershell
# 母艦から実行
# credentials.jsonをコピー
scp konoha:/root/credentials.json C:\Users\mana4\OneDrive\Desktop\manaos_integrations\credentials.json

# token.jsonをコピー
scp konoha:/root/token.json C:\Users\mana4\OneDrive\Desktop\manaos_integrations\token.json
```

#### 方法B: このはサーバーの画面共有経由

1. このはサーバーの画面共有（VNC等）にアクセス
2. 認証情報ファイルを開く
3. 内容をコピー
4. 母艦に新しいファイルとして保存

---

## 📝 注意事項

### credentials.jsonについて

- ✅ **共有可能**: 同じGoogle Cloudプロジェクトの認証情報なので、複数のマシンで使用可能
- ✅ **セキュリティ**: 機密情報なので、Gitにコミットしないこと

### token.jsonについて

- ⚠️ **注意**: token.jsonは通常、同じマシンで使用することを想定
- ✅ **動作確認**: コピー後、動作確認が必要
- 🔄 **再認証**: 動作しない場合は再認証が必要な場合あり

## 🔧 トラブルシューティング

### このはサーバーに接続できない

- SSHサービスが停止している可能性
- 接続方法を変更（X280経由、画面共有経由など）

### 認証情報ファイルが見つからない

- このはサーバー側でファイルの場所を確認
- 別の場所に保存されている可能性

### コピー後に動作しない

- token.jsonが期限切れの可能性
- 再認証が必要な場合あり
- `setup_google_drive.ps1`を実行して再認証

---

## 💡 ヒント

- **credentials.jsonのみコピー**: token.jsonが期限切れの場合、credentials.jsonだけコピーして再認証する方法も有効
- **バックアップ**: コピー前に既存のファイルをバックアップ
- **セキュリティ**: 認証情報ファイルは機密情報なので、適切に管理すること
