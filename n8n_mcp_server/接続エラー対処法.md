# n8n MCPサーバー 接続エラー対処法

**エラー:** "N8nへの接続エラー" / "サーバーに接続できませんでした"

---

## 🔍 原因の確認

### 1. n8nサーバーの状態確認

```powershell
# n8nサーバーが起動しているか確認
curl http://100.93.120.33:5678/healthz
```

**期待される結果:** `{"status":"ok"}`

### 2. MCPサーバーの起動確認

```powershell
cd C:\Users\mana4\OneDrive\Desktop\manaos_integrations
python -m n8n_mcp_server.server
```

**期待される動作:** MCPサーバーが起動する（エラーが出ない）

---

## 🛠️ 対処法

### 方法1: n8nサーバーが起動していない場合

このはサーバーでn8nを確認：

```bash
ssh konoha
docker ps | grep n8n
```

n8nが停止している場合は起動：

```bash
docker start trinity-n8n
```

### 方法2: APIキーが設定されていない場合

1. **n8nのWeb UIにアクセス**
   ```
   http://100.93.120.33:5678
   ```

2. **APIキーを作成**
   - Settings → API → Create API Key
   - APIキーをコピー

3. **設定ファイルに追加**
   ```powershell
   notepad "$env:USERPROFILE\.cursor\mcp.json"
   ```
   
   `N8N_API_KEY`の値をAPIキーに設定

4. **Cursorを再起動**

### 方法3: MCPサーバーのパスが正しくない場合

設定ファイルを確認：

```powershell
Get-Content "$env:USERPROFILE\.cursor\mcp.json" | ConvertFrom-Json | ConvertTo-Json -Depth 10
```

`cwd`が正しいパスを指しているか確認：
```
C:\Users\mana4\OneDrive\Desktop\manaos_integrations
```

### 方法4: Pythonモジュールが見つからない場合

```powershell
cd C:\Users\mana4\OneDrive\Desktop\manaos_integrations
python -c "import n8n_mcp_server.server; print('OK')"
```

エラーが出る場合は、パスの問題です。

## 💡 よくある問題

### 問題1: "モジュールが見つかりません"

**原因:** `cwd`のパスが正しくない

**対処:** 設定ファイルの`cwd`を確認

### 問題2: "認証エラー"

**原因:** APIキーが設定されていない、または無効

**対処:** n8nのWeb UIから新しいAPIキーを作成

### 問題3: "接続タイムアウト"

**原因:** n8nサーバーが起動していない

**対処:** このはサーバーでn8nを起動

---

3. ✅ MCPサーバーが起動できるか確認
4. ✅ Cursorを再起動
5. ✅ Cursorから再度試す
