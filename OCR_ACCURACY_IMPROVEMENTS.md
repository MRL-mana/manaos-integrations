# OCR精度向上の改善内容

## ✅ 実装した改善

### 1. 解像度向上
- **変更前**: zoom 3.0 (300 DPI相当)
- **変更後**: zoom 4.0 (400 DPI相当)
- **効果**: より高解像度で認識、細かい文字も読み取り可能

### 2. 前処理パターンの大幅拡張
- **既存**: base, 二値化(180), 反転, シャープ
- **追加**:
  - 複数コントラスト値 (1.5, 2.0, 2.5, 3.0)
  - 複数二値化閾値 (160, 180, 200)
  - 適応的閾値処理 (OpenCV使用、オプション)
  - ガウシアンブラー + シャープ
  - ノイズ除去強化 (MedianFilter size=5)
- **効果**: 様々な画像品質に対応、最適な前処理を自動選択

### 3. PSM範囲の拡大
- **既存**: [6, 4, 11]
- **追加**: [1, 3, 12, 13]
- **効果**: 表・帳票に適したPSMをより多く試行

### 4. 傾き補正（オプション）
- **機能**: 画像の傾きを検出して自動補正
- **条件**: auto=True の時のみ実行（処理時間考慮）
- **効果**: スキャン時の傾きを補正して精度向上

### 5. スコアリング改善
- **信頼度**: 非線形評価（高信頼度をより重視: conf^1.5）
- **数字比率**: 帳票では数字が多いため、数字比率をスコアに追加
- **文字化けペナルティ**: 文字化けの重みを2倍に強化
- **異常記号検出**: より多くの異常記号パターンを検出
- **効果**: より正確な結果を選択

### 6. Vision LLMプロンプト改善
- **詳細な指示**: 10項目の詳細な指示を追加
- **文字化け対策**: 不確実な文字の扱いを明確化
- **数値精度**: 小数点、カンマ、記号の保持を強調
- **効果**: Vision LLMの認識精度向上

## 📊 改善効果

### 処理パターン数
- **改善前**: 4前処理 × 3PSM = 12パターン
- **改善後**: 約15前処理 × 7PSM = 約105パターン（auto=True時）
- **効果**: より多くの組み合わせから最適解を選択

### 精度向上のポイント
1. **高解像度**: 細かい文字も認識可能（400 DPI）
2. **多様な前処理**: 様々な画像品質に対応
3. **適切なPSM選択**: 表構造に最適なPSMを自動選択
4. **傾き補正**: スキャン時の傾きを自動補正
5. **改善されたスコアリング**: より正確な結果を選択

## 🚀 使用方法

### 高精度OCR（推奨・デフォルト）
```bash
python pdf_to_excel_converter.py "input.pdf" "output.xlsx"
```

自動的に以下が有効になります：
- `auto=True` (複数前処理×PSM探索)
- `layout=True` (レイアウト情報取得)
- `lang="jpn+eng"` (日本語+英語)
- `zoom=4.0` (400 DPI)

## ⚠️ 注意事項

- **処理時間**: 前処理パターンが増えたため、処理時間が長くなります（約2-3倍）
- **メモリ使用量**: 高解像度（zoom 4.0）のため、メモリ使用量が増加します
- **OpenCV**: 適応的閾値処理と傾き補正にはOpenCVが必要です（無くても動作します）

## 🔧 カスタマイズ

### 処理速度優先
```python
from ocr_multi_provider import MultiProviderOCR
ocr = MultiProviderOCR()
result = ocr.recognize(
    "image.png",
    auto=False,  # 自動探索を無効化
    psm=6        # 固定PSM
)
```

### 最高精度（時間かかってもOK）
```python
result = ocr.recognize(
    "image.png",
    auto=True,
    psm_list=[1, 3, 4, 6, 11, 12, 13],  # 全PSM試行
    lang="jpn+eng"
)
```
