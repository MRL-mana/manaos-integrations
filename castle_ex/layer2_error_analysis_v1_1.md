## 1. l2_attribute（12件・正解0）— なぜ 0.0 か

### 問いの形
- 「X の [危険度/優先度/重要度/大きさ/役割] は？」

### 正解（Gold）の特徴
- **意図的に「ズラした」値**（データで決めた対応だけを返す想定）
  - 例: 車の危険度 → 「低い」、給油の優先度 → 「大きい」、オイルの危険度 → 「簡単」

### モデルの予測（Pred）
- **常識的な語**を返している
  - 危険度 → 「重要」
  - 優先度 → 「赤」「低い」
  - 重要度 → 「難しい」「簡単」（文脈で反対を出している）
  - 大きさ → 「大きい」「高い」

### 原因の整理
- **「このタスクでは、問いと値の対応はデータで固定。常識で答えない」** というルールが身体化していない。
- ベースの「自然な」属性推論（危険度→重要、優先度→赤 など）が強く、**データで決めたラベルをそのまま返す**動きになっていない。

### 次の一手（データ設計）
- **「属性タスクではラベルをそのまま返す」** を明示する例を大量に。
- 同じ属性名でも **複数パターン**（「低い」「大きい」「赤」「簡単」など）をランダムに割り当てた例を増やす。
- テンプレ: `Xの{属性}は？` → `Xの{属性}は{値}です。` の **対応ペアを +300〜500 件**（特に attribute / comparison）。

## 2. l2_comparison（12件中2正解・10誤答）

### 問いの形
- 「A と B、どっちが [早い/重要/効率的/危険]？」

### 正解（Gold）
- 「AとBでは{l2_a}の方が〜」「AとBでは{l2_b}の方が〜」（データでどちらを正解にするか固定）

### モデルの予測（Pred）
- **ほぼ常に逆側**（{l2_a} と {l2_b} を入れ替えた答え）
- スコア 0.47 付近 → 文面は似ているが「どっち」の選択が反対

### 原因の整理
- **「どっちが〜」の“順序”（A を選ぶか B を選ぶか）を、データのラベルに合わせて答える** ルールがまだ弱い。
- モデルは「常識で比較」しており、データ側の正解は「テンプレで決めた対応」のため、逆になりやすい。

### 次の一手（データ設計）
- **A/B のどちらを正解にするかをランダムに振り分けた比較問題** を大量に。
- 「Aの方が〜」「Bの方が〜」の **両パターン** を同じ質問形で出して、**データで決めた方だけが正解** である例を増やす（+300〜500 件で「選択の癖」を上書き）。

## 3. l2_part_whole（12件中5正解・7誤答）

### 問いの形
- 「X は Y の一部？」

### 傾向
- Gold が「はい。XはYに含まれます。」のとき、Pred は「いいえ。XはYに含まれません。」になりやすい。
- とくに **Y が「文脈」「システム」「空間」** など抽象的なとき誤答が多い。

### 原因の整理
- 「文脈」「システム」などが抽象的で、**どの具体物がその「全体」に含まれるか** をベースの常識だけでは一意に決められない。
- データでは「含まれる」にしているのに、モデルは「含まれない」と答える → **データで決めた対応（はい/いいえ）を優先する** 学習が足りない。

### 次の一手（データ設計）
- part_whole も **「データで決めた yes/no をそのまま返す」** 例を増やす。
- 抽象的な全体語（文脈・システム・空間）について、**含まれる/含まれないの両パターン** を意図的に混ぜて、データのラベルに従う例を追加。
