# n8nワークフロー再アクティベート手順

## 現在の状況
- n8nは起動中（http://localhost:5679）
- ワークフロー「ManaOS Image Generation Workflow」はインポート済み
- Webhookが404エラーで登録されていない

## 解決方法

### 方法1: Web UIから手動実行（最も簡単・推奨）

1. **ブラウザでn8nを開く**
   - http://localhost:5679 にアクセス
   - 既に開いている場合は、そのまま使用

2. **ワークフローを開く**
   - ワークフロー一覧から「ManaOS Image Generation Workflow」をクリック
   - または、ワークフローID `2ViGYzDtLBF6H4zn` を検索

3. **ワークフローを無効化→有効化**
   - ワークフローエディタの右上にあるトグルスイッチを確認
   - 現在「ON」（緑色）になっている場合は、一度「OFF」に切り替え
   - 2-3秒待つ
   - 再度「ON」に切り替え
   - これでWebhookが再登録されます

4. **確認**
   - Webhookノード（最初のノード）をクリック
   - 「Path」が `comfyui-generated` になっていることを確認
   - ワークフローが有効（緑のチェックマーク）になっていることを確認

### 方法2: API経由で実行

1. **新しいAPIキーを取得**
   - http://localhost:5679 を開く
   - 右上のユーザーアイコン → Settings → API
   - 「Create API Key」をクリック
   - 生成されたAPIキーをコピー

2. **APIキーを設定**
   ```powershell
   $env:N8N_API_KEY = "YOUR_API_KEY_HERE"
   ```

3. **ワークフローを再アクティベート**
   ```powershell
   cd C:\Users\mana4\OneDrive\Desktop\manaos_integrations
   python n8n_mcp_server/reactivate_workflow.py 2ViGYzDtLBF6H4zn
   ```

## 再アクティベート後の確認

1. **Webhook URLを確認**
   ```powershell
   python n8n_mcp_server/get_webhook_url.py 2ViGYzDtLBF6H4zn
   ```

2. **ワークフローを実行してテスト**
   ```powershell
   python n8n_mcp_server/execute_workflow.py 2ViGYzDtLBF6H4zn
   ```

3. **実行履歴を確認**
   - n8nのWeb UIで「Executions」タブを開く
   - 最新の実行結果を確認
