# LLM と 画像生成AI の役割分離（マナ向け）

**結論から言うと：「LLMでやる話」と「画像生成AIでやる話」は完全に別物。ここを混ぜると一生ハマる。**

---

## 正直な結論（Tell it like it is）

- **露骨さ・攻め表現が得意なのは LLM じゃない**
- **主戦場は画像生成AI（Stable Diffusion 系）**

LLM には限界がある：

- 学習段階で“言語的ブレーキ”が入りやすい
- ローカルでも「語彙がマイルド」になりがち
- 表現を続けると急にトーンが落ちる

だから、

| 担当 | 役割 |
|------|------|
| **LLM** | 設計・演出・プロンプト生成 |
| **画像生成AI** | 表現はここに丸投げ |

という**分業**が最強。

---

## 分野別

### LLM（文章・指示生成）で“攻めやすい系”

制限が弱め・表現が濁りにくい観点では：

- **LLaMA 系（派生・微調整）**
- **Mistral 系の派生**
- **Qwen 系のローカル版**

特徴：指示を曖昧にせず解釈しやすい、トーンが急に品行方正になりにくい、ロールプレイ・演出指示が得意。

⚠️ **LLM 単体で満足いく“露骨さ”を出そうとするのは非効率**（幻想）。

---

### 本命：画像生成AI（ここが勝ち筋）

**Stable Diffusion 系（ローカル）** が一択。理由：

- 完全ローカル ＝ 外部検閲なし
- モデル・LoRA・VAE で“方向性を物理的に変えられる”
- **表現の上限は「設定」と「素材」だけ**

重要になるのは：

- **モデル選び（思想が 9 割）**
- **LoRA の質**
- **Safety / Checker を完全に分離 or 排除**

ここが整うと「同じプロンプトなのに世界が変わる」現象が起きる。

---

## 勝ち構成（実戦向き）

```
[LLM]
  役割: 演出 / 指示 / プロンプト生成
  ローカル LLM（制限弱め）
  表現を言語化するだけ

        ↓

[画像生成AI]
  Stable Diffusion（完全ローカル）
  表現特化モデル
  Safety / Filter 一切なし
```

- **LLM に“露骨さ”を期待しない**
- **画像AI に全振り**

これが一番ストレスない。

---

## よくある勘違い（ハマりポイント）

- ❌ 「ローカル LLM なら何でもいける」
- ❌ 「GPT 系より強い LLM を探せば解決」
- ❌ 「プロンプトを工夫すれば限界突破」

**違う。** 限界突破するのは **モデルの思想 × ワークフロー分離**。

## マナ環境での回答（貼ってくれたら一気に詰める用）

### 1. 今の画像生成は WebUI？ComfyUI？

**→ ComfyUI。**

- `generate_50_mana_mufufu_manaos.py` / `gallery_api_server.py` が ComfyUI（`COMFYUI_URL`, デフォルト `http://localhost:8188`）に送信。
- ベースパスは `C:/ComfyUI`（`COMFYUI_BASE` で変更可）。
- WebUI（AUTOMATIC1111）はこのリポジトリでは使っていない。

### 2. GPU（5080）は余力ある？

コードからは判断できない。**余力があるかどうかはマナが「ある」と答えれば、別枠の実験用ワークフローや並列ジョブを組む前提にできる。**

### 3. 「完全に別枠で危険ゾーン作る」のアリ？

**アリなら、こう分ける設計が最短。**

| 枠 | 用途 | イメージ |
|----|------|----------|
| **通常世界** | 普段の生成・評価・学習 | 今の `--mode safe` / Gallery API の `mufufu_mode: false`、既存ネガ・Safety あり |
| **闇の実験室** | 表現を限界まで試す | 専用ワークフロー or 専用エンドポイント、ネガ最小限、Safety/Checker オフ、モデル・LoRA も別フォルダ推奨 |

実装イメージ（最短）：

- **設定の分離**
  - 例：`mufufu_config_safe.py`（通常）と `mufufu_config_lab.py`（闇用・ネガ最小・Checker なし）のように「実験室用」設定を分ける。
- **ワークフロー / エンドポイントの分離**
  - ComfyUI で「実験室用」ワークフローを別名で保存し、`generate_50_...` や Gallery API に `--profile lab` や `lab_mode: true` のようなフラグで切り替え。
- **出力先の分離**
  - 実験室は別フォルダ（例：`output_lab`）に吐く。評価UIで「通常だけ」「実験室だけ」を選べるようにする。

ここまで OK なら、**「通常世界」と「闇の実験室」を分ける設計**をガチで組める。ローカルの強みをちゃんと使う形になる。
