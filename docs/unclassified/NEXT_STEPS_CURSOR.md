## 📋 Cursor設定の手順

### ステップ1: Cursorの設定を開く

1. **Cursorを起動**
2. **設定を開く**
   - `Ctrl + ,` (Windows/Linux)
   - または、メニューから「Settings」を選択

### ステップ2: LLM設定を開く

1. **検索バーで「LLM」または「Model」を検索**
2. **「Features」または「AI」セクション**を開く
3. **「Model」または「LLM Provider」**を探す

### ステップ3: エンドポイントを設定

以下の設定を入力：

| 項目 | 値 |
|------|-----|
| **Provider** | `OpenAI Compatible` または `Custom` |
| **Base URL** | `http://localhost:1234/v1` |
| **API Key** | `lm-studio` または空欄 |
| **Model** | `qwen2.5-coder-7b-instruct` |

### ステップ4: 接続確認

1. **設定を保存**
2. **Cursorのチャットでテスト**
   - 簡単な質問をして、ローカルLLMが応答するか確認
   - 例：「Hello, can you help me?」

## 🎯 設定の詳細

### Base URLについて

- **正しい形式**: `http://localhost:1234/v1`
- **注意**: `/v1` が含まれていることを確認
- **確認方法**: ブラウザで `http://localhost:1234/v1/models` にアクセスして確認

### API Keyについて

- LM Studioは空欄でも動作する場合があります
- 動作しない場合は `lm-studio` を入力

### Modelについて

- **推奨モデル**: `qwen2.5-coder-7b-instruct`
- **他のモデル**: `nvidia/nemotron-3-nano`, `openai/gpt-oss-20b` なども使用可能

## 🔧 トラブルシューティング

### Cursorが接続できない場合

1. **LM Studioサーバーが起動しているか確認**
   ```powershell
   Invoke-WebRequest -Uri "http://localhost:1234/v1/models" -Method GET
   ```

2. **Base URLが正しいか確認**
   - `http://localhost:1234/v1` であることを確認
   - `/v1` が含まれていることを確認

3. **ファイアウォールの設定を確認**
   - ローカル接続がブロックされていないか確認

4. **Cursorを再起動**
   - 設定を変更した後、Cursorを再起動

### モデルが見つからない場合

1. **LM Studioでモデルが読み込まれているか確認**
   - 「Server」タブでモデルが「READY」になっているか確認

2. **モデル名が正しいか確認**
   - `qwen2.5-coder-7b-instruct` が正しいか確認
   - 大文字小文字を区別する場合があります

## 📊 パフォーマンス

### 期待される動作

- **レスポンス時間**: 数秒〜数十秒（モデルとハードウェアに依存）
- **品質**: コーディングタスクに特化したモデルを使用
- **オフライン**: インターネット接続不要

### 最適化のヒント

1. **軽量モデルを使用**
   - 7Bモデルで十分な場合が多い
   - 14Bモデルはより高品質だが、遅い

2. **コンテキスト長を調整**
   - 必要以上に長いコンテキストは避ける

3. **バッチ処理**
   - 複数のリクエストをまとめて処理
