# n8n MCPサーバー 接続エラー解決手順

**エラー:** "N8nへの接続エラー" / "サーバーに接続できませんでした"

---

## ✅ 確認済み

- ✅ n8nサーバー: 起動中（http://100.93.120.33:5678）
- ✅ MCPサーバーモジュール: 正常にインポート可能
- ✅ APIキー: 設定済み

---

## 🔧 解決手順

### ステップ1: Cursorを再起動

MCP設定を変更した場合は、Cursorを完全に再起動してください。

1. Cursorを閉じる
2. タスクマネージャーでCursorプロセスが残っていないか確認
3. Cursorを再起動

### ステップ2: MCP設定を確認

```powershell
Get-Content "$env:USERPROFILE\.cursor\mcp.json" | ConvertFrom-Json | ConvertTo-Json -Depth 10
```

**確認項目:**
- `command`: `"python"` になっているか
- `args`: `["-m", "n8n_mcp_server.server"]` になっているか
- `cwd`: 正しいパスを指しているか
- `env.N8N_API_KEY`: APIキーが設定されているか

### ステップ3: MCPサーバーを手動でテスト

```powershell
cd C:\Users\mana4\OneDrive\Desktop\manaos_integrations
python -m n8n_mcp_server.server
```

**期待される動作:** 
- エラーが出ずに起動する
- 標準入出力でMCPプロトコルを待機する

**エラーが出る場合:**
- エラーメッセージを確認
- 必要なパッケージがインストールされているか確認

### ステップ4: 環境変数を確認

```powershell
# 環境変数を確認
python -c "import os; print('N8N_BASE_URL:', os.getenv('N8N_BASE_URL', '未設定')); print('N8N_API_KEY:', '設定済み' if os.getenv('N8N_API_KEY') else '未設定')"
```

### ステップ5: CursorのMCPログを確認

Cursorの開発者ツールでMCPのログを確認：

1. `Ctrl + Shift + I` で開発者ツールを開く
2. ConsoleタブでMCP関連のエラーを確認

## 🛠️ よくある問題と対処法

### 問題1: "モジュールが見つかりません"

**原因:** Pythonのパスが正しくない

**対処:**
```powershell
# Pythonのパスを確認
python -c "import sys; print(sys.executable)"

# モジュールがインポートできるか確認
cd C:\Users\mana4\OneDrive\Desktop\manaos_integrations
python -c "import n8n_mcp_server.server; print('OK')"
```

### 問題2: "認証エラー"

**原因:** APIキーが無効または設定されていない

**対処:**
1. n8nのWeb UIにアクセス: http://100.93.120.33:5678
2. Settings → API → Create API Key
3. 新しいAPIキーをコピー
4. 設定ファイルに追加

### 問題3: "接続タイムアウト"

**原因:** n8nサーバーに接続できない

**対処:**
```powershell
# n8nサーバーの状態を確認
curl http://100.93.120.33:5678/healthz
```

### 問題4: MCPサーバーが起動しない

**原因:** 必要なパッケージがインストールされていない

**対処:**
```powershell
pip install mcp requests
```
