# 🚀 LM Studioサーバー起動クイックガイド

## ✅ 現在の状態

- ✅ LM Studio: インストール済み・起動中
- ⚠️  LM Studioサーバー: **手動起動が必要**

---

## 📋 3ステップで起動

### ステップ1: 「Server」タブをクリック

1. LM Studioのウィンドウを前面に表示
2. **上部のタブメニュー**を確認
3. 「**Server**」タブを**クリック**

```
┌─────────────────────────────────────────┐
│  [Chat] [Search] [Server] ← ここ！      │
└─────────────────────────────────────────┘
```

### ステップ2: モデルを選択

1. 「**Select a model to load**」を**クリック**
2. ダウンロード済みのモデルを選択
   - 推奨: **Qwen2.5-Coder-7B-Instruct**
   - まだダウンロードしていない場合は、先に「Search」タブでダウンロード

```
┌─────────────────────────────────────────┐
│  Select a model to load: [▼] ← ここ！   │
└─────────────────────────────────────────┘
```

### ステップ3: 「Start Server」をクリック

1. モデルを選択したら、「**Start Server**」ボタンを**クリック**
2. サーバーが起動すると、ボタンが「Stop Server」に変わります

```
┌─────────────────────────────────────────┐
│  [Start Server] ← ここをクリック！        │
└─────────────────────────────────────────┘
```

---

## ✅ 起動確認

サーバーが起動したら、以下で確認できます：

```powershell
.\check_running_status.ps1
```

**期待される結果：**
- ✅ LM Studioサーバー: 起動中 (`http://127.0.0.1:1234`)
- ✅ Unified API（LLMルーティング含む）: 起動中 (`http://127.0.0.1:9510`)

---

## 🎯 完全運用開始

すべてのサービスが起動したら：

1. **CursorでローカルLLMを使用**
   - Cursorの設定で `http://127.0.0.1:1234/v1` が設定済み
   - モデルを選択して使用開始

2. **MCPサーバー経由でLLMルーティングを使用**
   - CursorのMCP設定で `llm_routing_mcp_server` が有効
   - ツール経由でLLMルーティング機能を使用可能

3. **API経由でLLMルーティングを使用**
   - `http://127.0.0.1:9510/api/llm/route` でルーティングAPIを使用

---

## 🔧 トラブルシューティング

### 「Server」タブが見つからない
- LM Studioのバージョンが古い可能性があります
- 最新版に更新: https://lmstudio.ai/download

### モデルが選択できない
- モデルがダウンロードされていない可能性があります
- 「Search」タブでモデルを検索してダウンロード

### 「Start Server」ボタンが押せない
- モデルが選択されていない可能性があります
- 先にモデルを選択してください
