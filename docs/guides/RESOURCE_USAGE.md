# リソース使用状況と最適化

**更新日**: 2025-12-28

---

## 🔍 現在の状況

### GPUリソース競合

- **複数のollamaプロセスが同時実行中**
  - 特に`qwen2.5:72b`などの重いモデルが複数起動
  - GPUメモリが競合している可能性が高い

### 夜のルーチンの問題

- **使用モデル**: `qwen2.5:72b` (reasoningタスク)
- **問題**: GPU競合により処理が遅延
- **タイムアウト**: 30秒では不足（90秒推奨）

---

## 💡 解決策

### 1. タイムアウト時間の延長（即座に適用可能）

- **テストスクリプト**: 夜のルーチンのみ90秒に延長 ✅
- **APIエンドポイント**: Flaskのデフォルトタイムアウトを確認

### 2. モデルの軽量化（推奨）

夜のルーチンで使用するモデルを軽量化：

**オプションA**: `reasoning`タスクでも軽量モデルを使用
```python
# secretary_routines.py の _generate_daily_report() で
# task_type を "automation" に変更（qwen2.5:14bを使用）
```

**オプションB**: フォールバックを活用
- `qwen2.5:72b`がタイムアウトした場合、自動的に`qwen2.5:32b`にフォールバック
- 既に実装済み ✅

### 3. リソース管理の改善

- **Ollamaプロセスの管理**: 不要なプロセスを終了
- **GPU使用状況の監視**: リソース使用状況を確認してから実行

---

## 📊 推奨設定

### 夜のルーチン用設定

```yaml
# llm_routing_config.yaml
routing:
  reasoning:
    # 夜のルーチン専用（軽量化版）
    primary: "qwen2.5:32b"  # 72bから32bに変更
    fallback:
      - "qwen2.5:14b"       # さらに軽量
      - "llama3.1:8b"       # 最終フォールバック
```

### タイムアウト設定

- **テスト**: 90秒（夜のルーチンのみ）
- **API**: 60秒（Flaskのデフォルト）
