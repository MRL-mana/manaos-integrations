# 🔥 OpenWebUI APIキー設定について

## 📋 質問: APIキーを有効にする必要があるか？

### ✅ 回答: External Tools機能では通常不要

OpenWebUIの**External Tools機能**を使用する場合、**APIキーを有効にする必要は通常ありません**。

ブラウザからOpenWebUIにアクセスして「外部ツール」設定画面でTool Serverを登録・管理する場合は、セッション認証（ログイン）で十分です。

---

## 🔍 APIキーが必要な場合

APIキーが必要になるのは、以下のような場合です：

1. **外部からOpenWebUIのAPIを直接呼び出す場合**
   - スクリプトからOpenWebUIのAPIを呼び出す
   - 他のアプリケーションからOpenWebUIのAPIを呼び出す
   - CURLやPostmanなどのツールからOpenWebUIのAPIを呼び出す

2. **OpenWebUIのAPI経由でツールを使用する場合**
   - プログラムからOpenWebUIのAPI経由でツールを呼び出す

---

## 💡 現在の状況

現在、Tool Serverの登録状況を確認するために「外部ツール」設定画面に移動する必要があります。

1. **まず「外部ツール」設定画面に移動**
   - 左サイドバーの「外部ツール」をクリック
   - または、URLから直接: `http://localhost:3001/admin/settings/external-tools`

2. **Tool Serverの登録状況を確認**
   - `http://localhost:9503` が登録されているか確認
   - 接続状態が「Connected」になっているか確認

3. **必要に応じてAPIキーを有効化**
   - 現時点では、External Tools機能を使用するためにAPIキーを有効にする必要はありません
   - ただし、将来的に外部からOpenWebUIのAPIを直接呼び出す必要がある場合は、APIキーを有効にしてください

---

## 🔥 レミ先輩の推奨

### 優先度1: 「外部ツール」設定画面を確認

1. **まず「外部ツール」設定画面に移動**
   - これが最重要

2. **Tool Serverの登録状況を確認**
   - 登録されているか確認
   - 接続状態を確認

### 優先度2: APIキーの設定（将来的に必要になったら）

1. **APIキーを有効化**
   - 設定画面の「APIキーを有効にする」をONにする

2. **APIキーを作成**
   - 設定画面からAPIキーを作成
   - APIキーを保存（重要！）

3. **APIキーを使用**
   - 外部からOpenWebUIのAPIを呼び出す際にAPIキーを使用
