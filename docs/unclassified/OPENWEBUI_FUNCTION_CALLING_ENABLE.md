# 🔧 OpenWebUI Function Calling 有効化ガイド

## ❌ 問題

「高度なパラメータ」セクションの「Function呼び出し (Function calling)」が「デフォルト」になっている。

これが原因で、LLMがTool Serverのツールを認識しても、実際にツールを呼び出さない可能性があります。

## ✅ 解決方法

### Step 1: Function呼び出しを有効にする

1. **「高度なパラメータ」セクションを確認**
   - 設定画面で「高度なパラメータ」セクションを開く
   - 「Function呼び出し (Function calling)」を探す

2. **「Function呼び出し」をクリック**
   - 「Function呼び出し (Function calling)」の行をクリック
   - 右側の「デフォルト」をクリック

3. **設定を変更**
   - 「デフォルト」から「有効」または「自動」に変更
   - 可能な選択肢：
     - **有効 (Enabled)**: Function Callingを常に有効にする
     - **自動 (Auto)**: LLMが必要に応じて自動的にFunction Callingを使用する
     - **デフォルト (Default)**: モデルのデフォルト設定を使用（通常は無効）

4. **設定を保存**
   - 設定が自動保存される場合もあります
   - 保存ボタンがある場合は、それをクリック

### Step 2: チャットでテスト

1. **チャット画面に戻る**
   - 設定画面を閉じて、チャット画面に戻る

2. **テストメッセージを送信**
   ```
   dockerコンテナの状態を確認して
   ```

3. **Tool Serverが呼び出されることを確認**
   - LLMが`service_status`ツールを実際に呼び出す
   - ツールの実行結果が表示される

## 🔍 確認事項

### ✅ 設定が正しく完了しているかの確認

1. **Function呼び出しの設定**
   - 「Function呼び出し」が「有効」または「自動」になっているか確認
   - 「デフォルト」のままになっていないか確認

2. **チャットコントロールセクション**
   - 「Functionの選択」ドロップダウンでTool Serverのツールが表示されているか確認

3. **システムプロンプト**
   - Tool Serverの使用を促す内容が入力されているか確認（推奨）

## 🎯 使い方

### Function Callingが有効な場合

1. **自動でツールを呼び出す**
   - チャットでメッセージを送信（例：「dockerコンテナの状態を確認して」）
   - LLMが適切なツールを自動的に選択して呼び出す

2. **ツールの実行結果が表示される**
   - ツールの実行結果がチャット画面に表示される
   - エラーが発生した場合は、エラーメッセージが表示される

### Function Callingが無効な場合

1. **ツールが呼び出されない**
   - LLMがツールを使用しようとしても、実際には呼び出されない
   - ツールの説明だけが返される

## 🔥 レミ先輩のアドバイス

### ✅ やるべき

1. **Function呼び出しを「有効」または「自動」に設定**
   - 「デフォルト」のままでは動作しない可能性が高い

2. **設定後、必ずテスト**
   - チャットでテストメッセージを送信
   - Tool Serverが実際に呼び出されることを確認

3. **システムプロンプトでTool Serverの使用を促す**
   - LLMにTool Serverを使用するよう明確に指示

### ❌ やっちゃダメ

- Function呼び出しを「デフォルト」のままにする
- 設定後、テストせずに動作確認をしない
- システムプロンプトでTool Serverについて言及しない

## 📋 トラブルシューティング

### 問題: Function呼び出しを「有効」にしてもツールが呼び出されない

**原因**: 他の設定が原因の可能性があります

**解決方法**:
1. 「Functionの選択」ドロップダウンでTool Serverのツールが表示されているか確認
2. システムプロンプトでTool Serverの使用を促す内容を追加
3. Tool Serverが正しく起動しているか確認（http://localhost:9503）
4. ブラウザの開発者ツール（F12）でネットワークタブを確認

### 問題: 「有効」に設定できない

**原因**: 使用しているモデルがFunction Callingに対応していない可能性があります

**解決方法**:
1. 使用しているモデルがFunction Callingに対応しているか確認
2. 対応しているモデルに変更（例：`qwen2.5-coder-7b-instruct`）
3. より大きなモデルを試す（例：`qwen2.5-coder-14b-instruct`）
