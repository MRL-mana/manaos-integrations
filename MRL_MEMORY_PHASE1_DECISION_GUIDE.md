# Phase 1 (Read-only) 判定ガイド

## 📋 共有してほしい情報

Phase 1を起動したら、以下を共有してください：

### 1. SECURITYログ1行

```
SECURITY: auth=enabled, rate_limit=enabled, max_input=200000, pii_mask=enabled
```

### 2. ダッシュボード初期値

```
E2E p95: X.XXX秒
ゲート遮断率: XX%
矛盾検出率: XX%
スロット使用率の分散: X.XX
書き込み回数/分: X
```

---

## 🎯 判定結果

共有された情報に基づいて、以下を即座に判定します：

- **Go / 設定ミス / 止めるべき**
- どこを直すべきか
- Phase 2へ進む条件の最終確認

---

## 📊 判定基準（簡易版）

### SECURITYログ

- ✅ **Go**: すべてenabled
- ❌ **No-Go**: 1つでもdisabled or ログなし

### E2E p95

- ✅ **Go**: < 0.1秒
- ⚠️ **注意**: 0.1〜0.3秒
- ❌ **No-Go**: > 0.3秒 or 急増

### ゲート遮断率

- ✅ **Go**: 0〜80%
- ⚠️ **注意**: 80〜95%
- ❌ **No-Go**: 95〜100%

### 矛盾検出率

- ✅ **Go**: < 5%
- ⚠️ **注意**: 5〜10%
- ❌ **No-Go**: > 10% or 急増

### スロット使用率の分散

- ✅ **Go**: < 100
- ⚠️ **注意**: 100〜1000
- ❌ **No-Go**: > 1000

### 書き込み回数/分

- ✅ **Go**: 0
- ❌ **No-Go**: > 0（Read-onlyなのに書き込みが発生）

---

## 🔄 判定後のアクション

### Go判定の場合

1. Phase 1を継続（24〜48時間）
2. 定期的にダッシュボードを確認
3. Phase 2 Go条件を満たしたらPhase 2へ

### 注意判定の場合

1. 監視を強化（30分おきにダッシュボードを確認）
2. ログを確認
3. 問題が解決したらGo判定に変更
4. 問題が継続する場合はNo-Go判定に変更

### No-Go判定の場合

1. **即座にKill Switch**（`export FWPKM_WRITE_ENABLED=0`）
2. ログを確認
3. 設定を修正
4. 修正後、再度Phase 1を開始

---

## 🚀 Phase 2 Go条件（24〜48時間後）

Phase 1を24〜48時間回して、以下が満たされればPhase 2へ：

- [ ] SECURITYログが毎回正しい
- [ ] 重大エラー（5xx）が継続発生してない
- [ ] Read-onlyなのに書き込みが発生してない
- [ ] p95が安定（急増なし）
- [ ] 矛盾検出率が安定（急増なし）
- [ ] ゲート遮断率が95%貼り付きじゃない
