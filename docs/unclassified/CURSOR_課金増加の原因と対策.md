# Cursor課金増加の原因と対策

## 🔍 課金が増えた主な原因（実際の使用状況から判明）

### ⚠️ **実際の原因: Composer機能の過度な使用**

画像の使用状況レポートから、実際の原因が判明しました：

**On-Demand Usage（従量課金）: $23.61**
- **non-max-composer-1**: **6576.3万トークン**を使用し、**$23.61**の追加課金が発生
- これはCursorの**Composer機能**（AIがコードを生成・編集する機能）の使用によるもの
- 「non-max」は最大トークン数に達していないリクエストを意味します

**Included Usage（含まれる利用）:**
- composer-1: 3.3億トークン ($110.98) - Pro Plusプランに含まれる
- Auto: 1926万トークン ($9.25) - Pro Plusプランに含まれる

### 1. **Composer機能の使用頻度が高い**

Composer機能を頻繁に使用しているため、追加の従量課金が発生しています。

**影響:**
- Composer機能は通常のチャットよりも多くのトークンを使用
- コード生成・編集のたびに追加のLLM呼び出しが発生
- 6576.3万トークンが追加で使用され、$23.61の課金が発生

### 2. **プロンプト最適化エンジンの自動適用（可能性あり）**

`prompt-optimizer-rules.mdc`が`alwaysApply: true`で設定されており、追加のLLM呼び出しが発生している可能性があります。

### 3. **複数のMCPサーバーの動作**

複数のMCPサーバーが設定されており、これらが自動的に動作している可能性があります。

---

## ✅ 解決策（優先順位順）

### 解決策1: Composer機能の使用を減らす（最優先・推奨）

**Composer機能の使用を減らすことで、追加課金を削減できます。**

**具体的な対策:**

1. **通常のチャットを優先的に使用**
   - 簡単な質問やコードの説明は通常のチャットを使用
   - Composerは複雑なコード生成・編集が必要な場合のみ使用

2. **Composerの使用頻度を減らす**
   - 一度に大量のコードを生成するのではなく、小さな単位で生成
   - 既存のコードを手動で編集できる場合は、Composerを使わない

3. **Composerの設定を確認**
   - Cursorの設定でComposerの動作を確認
   - 必要に応じて、Composerの自動実行を無効化

**効果:**
- **$23.61の追加課金を削減可能**
- Composer機能の使用を50%減らせば、約$11.80の削減

### 解決策2: プロンプト最適化ルールを無効化

プロンプト最適化の自動適用を無効化します：

**手順:**
1. `c:\Users\mana4\Desktop\.cursor\rules\prompt-optimizer-rules.mdc`を開く
2. `alwaysApply: true`を`alwaysApply: false`に変更
3. または、ファイル名を変更して無効化（例: `prompt-optimizer-rules.mdc.disabled`）

**効果:**
- プロンプト最適化による追加のLLM呼び出しを停止
- 課金を削減

### 解決策3: Cursorのモデル設定を確認・変更

Cursorで使用しているモデルを確認し、必要に応じて変更します。

**確認方法:**
1. Cursorの設定を開く（`Ctrl + ,`）
2. 「Model」または「AI Model」を検索
3. 使用しているモデルを確認
4. 必要に応じて、より安価なモデル（Claude Sonnet、GPT-3.5など）に変更

**注意:**
- Composer機能で使用されるモデルも確認してください
- より安価なモデルに変更することで、追加課金を削減できます

### 解決策4: 使用量の監視と制限設定

Cursorの使用量を監視し、制限を設定します。

**確認方法:**
1. Cursorのアカウント設定を開く
2. 「Usage」または「使用量」を確認
3. 「On-Demand Usage」の制限を設定（例: $20/月）
4. 制限に達したら通知を受け取る設定にする

## 🛠️ 即座に実行できる対策

### ステップ1: プロンプト最適化ルールを無効化

```powershell
# プロンプト最適化ルールを無効化
Rename-Item -Path "$env:USERPROFILE\Desktop\.cursor\rules\prompt-optimizer-rules.mdc" -NewName "prompt-optimizer-rules.mdc.disabled"
```

### ステップ2: Cursorを再起動

プロンプト最適化ルールを無効化した後、Cursorを完全に再起動してください。

### ステップ3: 使用量を確認

Cursorのアカウント設定で使用量を確認し、課金が減ったか確認してください。
