# アンサンブル修正ガイド

## 概要

複数のLLMモデルで同じセルを修正し、最良の結果を選択する方式です。
複数のモデルの意見を統合することで、より高い精度を実現します。

## 利点

- ✅ **より高い精度**: 複数モデルの意見を統合
- ✅ **エラー削減**: 1つのモデルの誤りを他のモデルが補正
- ✅ **信頼性向上**: 複数モデルが一致した結果はより信頼できる

## 使用方法

### 基本的な使い方

```powershell
# 環境変数を設定
$env:USE_LM_STUDIO="1"
$env:MANA_OCR_USE_LARGE_MODEL="1"

# アンサンブル修正を実行（自動モデル選択）
python excel_llm_ensemble_corrector.py input.xlsx output.xlsx --verbose
```

### モデルを指定する場合

```powershell
python excel_llm_ensemble_corrector.py input.xlsx output.xlsx --models qwen2.5-coder-32b-instruct qwen2.5-coder-14b-instruct openai/gpt-oss-20b --verbose
```

## 動作原理

1. **複数モデルで修正**: 各モデルで同じセルを修正
2. **結果の比較**: 各モデルの修正結果を比較
3. **最良の結果を選択**:
   - 最も多くのモデルが一致した結果
   - 同数の場合は、最も長い結果（より詳細な修正）
   - 元のテキストとの差が大きい結果（より積極的な修正）

## 使用モデル

### 自動選択（推奨）
- 利用可能なモデルを自動検出
- 最大3モデルまで使用
- 優先順位: 32B > 14B > 20B > 7B

### 手動指定
- 使用するモデルを明示的に指定
- 複数のモデルを指定可能

## 処理時間

- 複数モデルを使用するため、処理時間は長くなります
- ただし、精度が大幅に向上します
- 各モデルで並列処理を検討中

## 期待される効果

- **精度向上**: 単一モデルより10-20%精度向上
- **エラー削減**: 誤修正の削減
- **信頼性**: 複数モデルが一致した結果はより信頼できる

## 注意事項

- 複数モデルを使用するため、メモリ使用量が増加します
- 処理時間は使用モデル数に比例して長くなります
- GPUメモリが16GB以上推奨です
