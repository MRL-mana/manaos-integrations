# GitHubプライバシー設定について

⚠️ **重要**: このプロジェクトは**プライベートリポジトリ**として管理してください。

## 🔒 プライバシー設定

### リポジトリ作成時

新しいリポジトリを作成する際は、必ず**プライベート**として作成してください：

```bash
python setup_github_repo.py
```

（自動的にプライベートリポジトリとして作成されます）

### 既存リポジトリの確認

リポジトリのプライバシー設定を確認：

```bash
python check_repo_privacy.py
```

### 公開リポジトリをプライベートに変更

もし誤って公開リポジトリとして作成してしまった場合：

1. GitHubのリポジトリページにアクセス
2. **Settings** → **General** → **Danger Zone**
3. **"Change repository visibility"** → **"Change to private"**

## 🛡️ 機密情報の保護

### .gitignoreで除外されているファイル

以下のファイルは自動的に除外されています：

- `.env` - 環境変数（APIキーなど）
- `credentials.json` - 認証情報
- `token.json` - トークンファイル
- `*.db` - データベースファイル
- `*.pem`, `*.key` - 秘密鍵
- `auth.db` - 認証データベース

### 確認事項

コミット前に以下を確認してください：

1. **APIキーやトークンがコードに含まれていないか**
2. **認証情報ファイルがコミットされていないか**
3. **個人情報が含まれていないか**

### 誤ってコミットしてしまった場合

機密情報を誤ってコミットしてしまった場合：

1. **すぐにGitHubのトークン/キーを無効化**
2. **Git履歴から削除**（`git filter-branch`または`git filter-repo`）
3. **新しいトークン/キーを発行**

## 📋 チェックリスト

- [ ] リポジトリがプライベートに設定されている
- [ ] `.env`ファイルが`.gitignore`に含まれている
- [ ] 認証情報ファイルがコミットされていない
- [ ] APIキーがコードに直接書かれていない
- [ ] 個人情報が含まれていない

## 🔍 確認コマンド

```bash
# リポジトリのプライバシー設定を確認
python check_repo_privacy.py

# コミットされるファイルを確認
git status

# .gitignoreで除外されているか確認
git check-ignore -v .env
```
